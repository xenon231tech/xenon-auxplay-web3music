# Contributing to Auxplay

Thank you for your interest in contributing to Auxplay! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and inclusive
- Focus on code quality
- Help others learn and grow
- Report issues constructively

## Getting Started

1. **Fork the repository**
   ```bash
   git clone https://github.com/YOUR_USERNAME/auxplay.git
   cd auxplay
   ```

2. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make changes**
   - Keep commits atomic and well-documented
   - Follow the coding standards
   - Test your changes

4. **Submit a Pull Request**
   - Describe what you changed and why
   - Reference any related issues
   - Ensure all tests pass

## Coding Standards

### JavaScript
- Use ES6+ syntax
- Use meaningful variable names
- Add comments for complex logic
- Follow airbnb style guide

### HTML/CSS
- Use semantic HTML
- Follow BEM naming convention
- Keep CSS modular
- Optimize for mobile

### Commit Messages
```
feat: Add new feature
fix: Fix specific bug
docs: Update documentation
style: Format or refactor code
test: Add or update tests
chore: Update dependencies
```

## Reporting Issues

### Bug Reports
Include:
- Browser and OS version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

### Feature Requests
Describe:
- The use case
- Expected behavior
- Why it's valuable
- Implementation ideas

## Review Process

1. Submit PR with clear description
2. Wait for maintainer review
3. Address feedback if requested
4. PR merged when approved

## Development Setup

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

# Run tests
npm test

# Lint code
npm run lint
```

## Testing

- Write tests for new features
- Ensure all tests pass before PR
- Aim for 80%+ code coverage
- Test on multiple browsers

- Join our Discord community
